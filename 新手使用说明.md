本文面向第一次接触本项目的使用者、课程助教和二次开发者。目标是让你能从零启动系统、完成知识库导入、使用问答和学习计划，并知道如何通过 API、前端函数和后端 Python 函数调用核心能力。

## 1. 这个系统能做什么

Local RAG 是一个本地/私有知识库学习助手。用户登录后，可以上传个人资料，基于公共知识库和个人知识库提问；系统会返回答案、引用来源、推荐追问，并持续生成学习画像。学习画像中的薄弱概念可以继续生成复习计划、练习题和综合试卷。

核心能力包括：

- 账号注册、登录和用户数据隔离。
- 上传 `.txt`、`.md`、`.pdf` 文档作为个人知识库。
- 使用 `data/raw` 中的系统公共知识库。
- 通过 ChromaDB 做向量检索。
- 通过 Ollama 生成 embedding，默认使用 `bge-m3`。
- 通过 Ollama、DeepSeek 或 OpenAI-compatible 模型生成回答。
- 自动提取核心知识点、生成追问、记录长期记忆。
- 自动维护学习画像、薄弱概念、知识漏洞和复习建议。
- 自动生成学习计划、选择题、填空题、综合试卷，并保存学习历史。

## 3. 运行前准备

需要先准备这些软件：

- Python 3.11+
- Anaconda 或 Miniconda
- Ollama
- Node.js 18+

推荐先拉取两个 Ollama 模型：

```bash
ollama pull qwen2.5:3b
ollama pull bge-m3
```

说明：

- `qwen2.5:3b` 是默认本地聊天模型。
- `bge-m3` 是默认 embedding 模型，向量检索必须用到。
- 即使选择 DeepSeek 或自定义 OpenAI-compatible 模型做聊天，embedding 仍然依赖 Ollama。

## 4. 第一次初始化

进入项目根目录：

```bash
cd /系统路径/RAG
```

创建并激活 conda 环境：

```bash
conda env create -f environment.yml
conda activate rag
```

如果 conda 下载较慢，也可以先安装 ChromaDB，再补装其他 pip 依赖：

```bash
conda install -c conda-forge chromadb
pip install -r requirements.txt
```

创建本地配置文件：

```bash
cp .env.example .env
```

编辑 `.env`，新手通常只需要改这两项：

```env
AUTH_SECRET=换成一个固定的长随机字符串
DEEPSEEK_API_KEY=如果使用DeepSeek则填写你的key
```

其他路径通常不需要改，系统会自动推导：

```env
CHROMA_PATH=项目根目录/db/chroma
DATA_DIR=项目根目录/data/raw
USER_DATA_ROOT=项目根目录/data/users
```

## 6. 第一次网页使用流程

### 6.1 注册和登录

打开 `http://127.0.0.1:5173` 后，会先看到登录/注册界面。

操作步骤：

1. 点击注册。
2. 输入邮箱和至少 8 位密码。
3. 注册成功后进入系统。
4. 以后使用同一邮箱和密码登录即可。

登录 token 会保存在浏览器 `localStorage`。如果 token 过期，系统会提示重新登录。

### 6.2 说明页

进入系统后可以先打开左侧导航的“说明”。该页解释推荐使用流程：

```text
上传资料 -> 调整检索范围 -> 按主题提问 -> 查看引用与画像 -> 生成练习或综合试卷 -> 提交答卷 -> 回看学习历史
```

### 6.3 知识库页

打开“知识库”页面，可以看到两类文件：

- 系统知识库：来自 `data/raw`，所有用户可读，普通用户不能删除。
- 我的知识库：当前登录用户上传的个人文件，保存在 `data/users/<user_id>`。

上传个人文件：

1. 点击“导入新文档”。
2. 选择 `.txt`、`.md` 或 `.pdf`。
3. 后端会自动解析、切片、向量化并写入个人向量库。
4. 上传完成后可在线预览。

删除个人文件：

1. 在“我的知识库”文件列表中点击“删除”。
2. 后端会删除原文件，并删除该文件对应的向量切片。

### 6.4 用户设置页

打开“用户设置”页面，可以配置：

- 文本块召回数量 `top_k`：默认 4，范围 1 到 8。
- DeepSeek thinking：可对 `deepseek-v4-flash` 和 `deepseek-v4-pro` 单独开启或关闭。
- 检索范围：
  - 全部知识库
  - 仅个人 RAG 库
  - 仅系统知识库
- 搜索文件：可以勾选参与检索的具体文件。
- 私人 OpenAI API 格式模型：
  - API Key
  - Base URL
  - 模型名称
  - temperature、top_p、max_tokens、timeout
  - 是否用于问答模块
  - 是否用于学习计划模块

注意：自定义 API key 会加密保存在后端，不会在页面回显明文。

### 6.5 问答页

打开“问答”页面后，可以：

- 新建多个对话窗口。
- 给当前对话重命名。
- 切换聊天模型。
- 输入问题并发送。
- 点击推荐追问继续问。
- 点击回答下方的引用来源，查看原文或 PDF 页码。
- 查看右侧学习画像。
- 点击薄弱概念进入学习计划。

每次提问后，后端会执行完整 RAG 流程：

```text
用户问题
-> 根据聊天历史改写成独立问题
-> 提取核心知识点
-> 构造增强检索 query
-> 用 bge-m3 生成 query embedding
-> 检索系统知识库和个人知识库
-> 检索用户长期记忆
-> 构造上下文
-> 调用聊天模型生成回答
-> 生成 3 个追问
-> 抽取值得长期记住的用户事实
-> 分析知识漏洞和误区
-> 更新学习画像
-> 返回答案、引用、追问、学习诊断和画像
```

### 6.6 学习画像

学习画像显示在问答页右侧。

主要内容：

- 抽象评价：对当前学习状态的总体概括。
- 薄弱概念：分数越高表示越薄弱。
- 学习模块：系统推荐继续复习的内容。
- 薄弱历史：曾经出现过的薄弱点。

可以进行的操作：

- 点击“刷新”同步后端画像。
- 点击薄弱概念生成对应学习计划。
- 删除某个画像标签或历史条目。

### 6.7 学习计划页

打开“学习计划”页面，可以：

- 输入一个概念生成专题学习计划。
- 从学习画像点击薄弱概念自动进入计划。
- 点击“综合试卷”基于完整学习画像组卷。
- 选择出题模型。
- 作答选择题、填空题、问答题。
- 提交当前答卷。
- 删除某个学习模块。
- 删除未完成学习计划。

提交答卷后：

- 系统会计算正确数和得分。
- 如果不及格，会提高对应概念薄弱度。
- 如果优秀，会降低对应概念薄弱度。
- 如果及格，会保持当前状态。
- 系统会自动生成下一张对应难度的试卷。

### 6.8 学习历史页

打开“学习历史”页面，可以查看已完成试卷：

- 主题
- 难度
- 得分
- 每道题的答案
- 判题反馈

也可以删除某条已完成学习记录。

## 7. 系统知识库维护

系统公共知识库放在：

```text
data/raw/
```

支持文件格式：

```text
.txt
.md
.pdf
```

添加公共资料后，需要重建公共向量索引：

```bash
cd /系统路径/RAG
conda activate rag
python -m app.ingest
```

这个命令会：

1. 扫描 `data/raw`。
2. 读取 TXT/MD/PDF。
3. 清洗文本。
4. 按 `CHUNK_SIZE` 和 `CHUNK_OVERLAP` 切片。
5. 调用 Ollama 的 embedding 接口。
6. 重建公共 ChromaDB collection。

默认公共 collection 名：

```text
rag_demo
```

个人知识库 collection 名：

```text
rag_demo_<user_id>
```

长期记忆 collection 名：

```text
user_memory_<user_id>
```

## 9. HTTP API 调用方法

下面示例默认后端地址是：

```bash
BASE_URL=http://127.0.0.1:8000
```

### 9.1 注册

```bash
curl -X POST "$BASE_URL/api/auth/register" \
  -H "Content-Type: application/json" \
  -d '{"email":"test@example.com","password":"12345678"}'
```

返回：

```json
{
  "token": "...",
  "user": {
    "id": "user_xxx",
    "email": "test@example.com"
  }
}
```

### 9.2 登录

```bash
curl -X POST "$BASE_URL/api/auth/login" \
  -H "Content-Type: application/json" \
  -d '{"email":"test@example.com","password":"12345678"}'
```

登录后，把返回的 token 和 user_id 保存下来：

```bash
TOKEN="替换成返回的token"
USER_ID="替换成返回的user.id"
```

后续需要登录态的接口都要带：

```bash
-H "Authorization: Bearer $TOKEN"
```

### 9.3 查看当前用户

```bash
curl "$BASE_URL/api/auth/me" \
  -H "Authorization: Bearer $TOKEN"
```

### 9.4 查看可用模型

```bash
curl "$BASE_URL/api/models" \
  -H "Authorization: Bearer $TOKEN"
```

### 9.5 读取用户设置

```bash
curl "$BASE_URL/api/settings" \
  -H "Authorization: Bearer $TOKEN"
```

### 9.6 更新用户设置

```bash
curl -X PUT "$BASE_URL/api/settings" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "top_k": 4,
    "deepseek_thinking_enabled": true,
    "deepseek_thinking": {
      "deepseek-v4-flash": true,
      "deepseek-v4-pro": true
    },
    "search_scope": "all",
    "selected_files": [],
    "custom_openai": {
      "base_url": "https://api.openai.com/v1",
      "model_name": "",
      "enabled_for_chat": false,
      "enabled_for_review": false
    }
  }'
```

`search_scope` 可选值：

```text
all
personal
system
```

`selected_files` 示例：

```json
[
  { "scope": "system", "source": "demo1.md" },
  { "scope": "personal", "source": "my-note.pdf" }
]
```

### 9.7 测试自定义 OpenAI-compatible 模型

先保存 API key、Base URL 和模型名称，再调用：

```bash
curl -X POST "$BASE_URL/api/settings/test-openai" \
  -H "Authorization: Bearer $TOKEN"
```

### 9.8 查看知识库文件

```bash
curl "$BASE_URL/api/files" \
  -H "Authorization: Bearer $TOKEN"
```

### 9.9 上传个人知识库文件

```bash
curl -X POST "$BASE_URL/api/upload" \
  -H "Authorization: Bearer $TOKEN" \
  -F "file=@/path/to/your-file.pdf"
```

预览个人文件：

```bash
curl "$BASE_URL/api/files/my-note.pdf/preview?scope=personal" \
  -H "Authorization: Bearer $TOKEN"
```

预览系统文件：


```bash

```

```text
/api/files/demo.pdf/raw?scope=system&access_token=<token>
```

### 9.12 删除个人文件

```bash

  -H "Authorization: Bearer $TOKEN"
```

### 9.13 重建当前用户个人索引

```bash
curl -X POST "$BASE_URL/api/ingest" \
  -H "Authorization: Bearer $TOKEN"
```

### 9.14 提问

```bash
curl -X POST "$BASE_URL/api/query" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d "{
    \"question\": \"这份资料主要讲了什么？\",
    \"user_id\": \"$USER_ID\",
    \"history\": [],
    \"chat_model\": \"qwen2.5:3b\",
    \"top_k\": 4,
    \"search_scope\": \"all\",
    \"selected_files\": [],
    \"deepseek_thinking_enabled\": true,
    \"custom_model_enabled\": false
  }"
```

带历史的 `history` 示例：

```json
[
  { "role": "user", "content": "先介绍一下PID控制。" },
  { "role": "assistant", "content": "PID控制由比例、积分、微分三部分组成..." }
]
```

返回字段：

```json
{
  "answer": "回答正文",
  "chunks": [
    {
      "source": "来源文件",
      "scope": "system 或 personal",
      "page": 1,
      "distance": 0.1234,
      "text": "命中的文本块"
    }
  ],
  "followups": ["追问1", "追问2", "追问3"],
  "core_terms": ["核心词1", "核心词2"],
  "learning_insight": {},
  "profile": {}
}
```

### 9.15 读取学习画像

```bash
curl "$BASE_URL/api/profile/$USER_ID" \
  -H "Authorization: Bearer $TOKEN"
```

### 9.16 删除画像标签

```bash
curl -X POST "$BASE_URL/api/profile/$USER_ID/tag/delete" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "category": "concept_mastery",
    "value": "PID控制"
  }'
```

`category` 常用值：

```text
concept_mastery
recommended_review
weak_history
knowledge_gaps
misconceptions
```

### 9.17 手动调整概念薄弱度

```bash
curl -X POST "$BASE_URL/api/profile/$USER_ID/weakness/update" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "concept": "PID控制",
    "delta": -1,
    "reason": "用户已完成复习并掌握"
  }'
```

说明：

- `delta > 0` 会增加薄弱度。
- `delta < 0` 会降低薄弱度。
- 实际每次调整步长为 `0.2`。

### 9.18 清除长期记忆和学习画像

```bash
curl -X DELETE "$BASE_URL/api/user/$USER_ID/history" \
  -H "Authorization: Bearer $TOKEN"
```

### 9.19 清除所有用户历史

```bash
curl -X DELETE "$BASE_URL/api/user/$USER_ID/all-history" \
  -H "Authorization: Bearer $TOKEN"
```

说明：会清除问答长期记忆、学习画像、学习计划和学习历史，但不清空知识库文件。

### 9.20 查看复习历史

```bash
curl "$BASE_URL/api/review/history/$USER_ID" \
  -H "Authorization: Bearer $TOKEN"
```

### 9.21 生成专题学习计划

```bash
curl -X POST "$BASE_URL/api/review/plan" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d "{
    \"user_id\": \"$USER_ID\",
    \"topic\": \"PID控制\",
    \"weak_score\": 0.6,
    \"difficulty\": 3,
    \"mode\": \"topic\",
    \"chat_model\": \"qwen2.5:3b\",
    \"deepseek_thinking_enabled\": true,
    \"custom_model_enabled\": false
  }"
```

## 12. 后端业务主流程说明

### 12.1 上传文件流程

入口：

```text
POST /api/upload
```

业务链路：

```text
校验登录用户
-> 校验文件后缀
-> 保存到 data/users/<user_id>/
-> 调用 index_file()
-> build_records() 解析文件
-> split_text() 切片
-> OllamaClient.embed() 向量化
-> get_collection(user_id) 获取个人 collection
-> collection.add() 写入 ChromaDB
```

### 12.2 问答流程

入口：

```text
POST /api/query
```

核心函数：

```text
resolve_chat_options()
extract_core_terms()
build_retrieval_query()
OllamaClient.embed()
get_collection()
diverse_results()
build_context()
chat_with_options()
generate_followups()
analyze_learning_state()
update_learning_profile()
ProfileStore.save()
```

### 12.3 学习画像流程

入口：

```text
问答完成后自动触发
```

业务链路：

```text
本轮问题、回答、核心词、引用片段、长期记忆、旧画像
-> analyze_learning_state()
-> 得到 topic、summary、weak_concepts、knowledge_gaps、misconceptions、recommended_review
-> update_learning_profile()
-> 更新 concept_mastery、weak_history、recent_insights、recent_questions
-> generate_abstract_evaluation()
-> ProfileStore.save()
```

薄弱度规则：

- `score` 越高表示越薄弱。
- `score >= 0.55` 状态为 `weak`。
- `0 < score < 0.55` 状态为 `learning`。
- `score <= 0` 或状态为 `familiar/mastered` 时会从活跃薄弱概念中清除。
- 每次自动或手动调整步长为 `0.2`。

### 12.4 学习计划流程

入口：

```text
POST /api/review/plan
POST /api/review/answer
POST /api/review/complete
```

生成计划链路：

```text
校验用户
-> resolve_chat_options()
-> 根据 mode 判断专题计划或综合试卷
-> collect_review_questions() 收集历史题目避免重复
-> generate_review_plan() 或 generate_comprehensive_review_plan()
-> normalize_review_plan() 补齐结构
-> ReviewStore.add() 保存
```

提交答卷链路：

```text
提交每道题答案
-> judge_review_answer()
-> 更新 session.answers 和 score
-> complete_review()
-> 计算 correct_count、result、next_difficulty
-> 根据结果更新学习画像薄弱度
-> ReviewStore.update()
```

结果规则：

```text
答对 >= 4 题：excellent，下一张难度 +1
答对 = 3 题：passed，难度不变
答对 <= 2 题：failed，下一张难度 -1
```

```bash
ollama pull bge-m3
```

确认 Ollama 正在运行后重试。

### 16.2 上传文件成功但索引失败

常见原因：

- Ollama 未运行。
- `bge-m3` 没拉取。
- PDF 无可抽取文本。
- ChromaDB 环境异常。

处理：

```bash
ollama pull bge-m3
python -m app.ingest
```

### 16.3 问答提示知识库中暂无相关文档片段

可能原因：

- 没有上传个人文档。
- 系统知识库没有重建索引。
- 设置页只勾选了不相关文件。
- 检索范围选成了“仅个人”或“仅系统”。

处理：

1. 检查知识库文件是否存在。
2. 检查设置页的检索范围和勾选文件。
3. 对系统知识库执行 `python -m app.ingest`。
4. 个人文件可重新上传或点击接口 `/api/ingest` 重建。

### 16.4 前端构建有 chunk 过大提醒

构建时可能看到：

```text
Some chunks are larger than 500 kB after minification
```

这是 Vite 的体积提醒，不代表构建失败。主要原因是 PDF.js worker 和 KaTeX 字体较大。后续可以通过动态导入 PDF 预览组件来优化。

### 16.5 `npm audit` 提示 moderate vulnerability

这来自 npm 依赖审计。不要盲目执行 `npm audit fix --force`，因为它可能升级大版本并破坏构建。建议先评估影响，再单独升级相关依赖。

### 16.6 登录后前端行为异常

可以清理浏览器当前站点的 localStorage 后重新登录。

常用 localStorage key：

```text
local_rag_auth_token
local_rag_auth_user
local_rag_conversations_<user_id>
local_rag_active_conversation_<user_id>
local_rag_profile_<user_id>
local_rag_selected_chat_model
```

### 16.7 PDF 预览加载慢

PDF 首次预览会加载 PDF.js worker、字体和 cmap 资源，较大 PDF 会更慢。等待加载完成即可。

## 17. 新手推荐使用清单

第一次使用建议按这个顺序：

1. 安装 Ollama、Node.js、Conda。
2. 拉取 `qwen2.5:3b` 和 `bge-m3`。
3. 创建 conda 环境。
4. 复制并编辑 `.env`。
5. 启动后端。
6. 启动前端。
7. 注册账号。
8. 打开“知识库”，上传一份 `.md` 或 `.pdf`。
9. 打开“用户设置”，确认检索范围为“全部知识库”。
10. 打开“问答”，围绕文档提问。
11. 点击引用查看原文。
12. 查看右侧学习画像。
13. 点击薄弱概念生成学习计划。
14. 完成答卷。
15. 打开“学习历史”复盘。
